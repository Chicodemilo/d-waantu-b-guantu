# Teams and Context Engineering

User prefers using Claude Code Teams (separate panes, teammates communicate directly) over manually sequenced subagent waves for multi-agent builds.

**Why:** Teams allow all specialists to be live simultaneously with the lead orchestrating dependencies via instructions, not manual wave launches. User sees this as "context engineering" — giving each teammate the right context at the right time.

**How to apply:** For future multi-agent work, suggest Teams with orchestration instructions baked in. Lead designs the context flow: what each teammate needs to know, what to wait for, what to signal when done. Don't manually sequence waves when Teams can handle coordination through teammate messaging.
