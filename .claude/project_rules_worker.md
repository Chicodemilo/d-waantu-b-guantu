# Project Rules — Workers

> Project-specific rules for all workers. This file is NOT overwritten by deploy.

## Identity at Spawn

Read the **Identity (REQUIRED — do not skip)** section in `.claude/agents/worker.md` first. Five steps: identify via `POST /api/agents/identify`, cache your `agent_id`, the TL writes your session marker (subagents can't touch `.claude/`), read your memory dir, follow the spawn-time read order. Without identity setup your work doesn't attribute correctly and may not appear in the dashboard at all.

## Alembic Migrations

- Autogenerate always detects spurious `drop_index` ops on the `error_logs` table (`ix_error_logs_created_at`, `ix_error_logs_source`). **Remove these** from every generated migration before running.
- Always add `server_default=sa.text('...')` when adding NOT NULL columns to tables with existing rows, plus an explicit `op.execute("UPDATE ...")` to backfill.
- Review every autogenerated migration before running — don't blindly trust the diff.

## Code Headers

Every `.py` file must have the standard header block. When modifying a file, update `Last Modified` to today's date. When the file's purpose or callees change, update those fields too.

## Gate Pattern (Doc Gates)

Doc gates follow a 4-point wiring pattern. When adding a new `force_*_md` gate:
1. Column in `app/models/project.py`
2. Fields in `app/schemas/project.py` (ProjectCreate, ProjectUpdate, ProjectRead)
3. Entry in `_DOC_GATES` list in `app/routers/projects.py` (gate-status endpoint)
4. Entry in the doc gates loop in `app/services/sprint.py` (sprint close enforcement)

Also add the filename to `_DOC_FILES` in `app/routers/projects.py` and to the seed demo in `app/services/seed_demo.py`.

## Don't Commit

Workers do NOT commit. Make changes, verify via API or tests, and report back to Archie. He reviews and commits.

## API Verification

After any backend change, verify with a real `curl` call against `http://localhost:8000`. Don't report work as done without confirming the API returns the expected response.

## Testing Patterns (Backend)

### Fixtures — use the factories in conftest.py
All factory fixtures accept `**overrides` to customize fields. Use them instead of raw POST calls when you don't need to control every field:
- `make_project()` — auto-increments prefix/name
- `make_agent()` — auto-increments name, unique api_key
- `make_epic(project_id=...)` — auto-creates project if omitted
- `make_sprint(project_id=..., epic_id=...)` — auto-creates both if omitted
- `make_ticket(project_id=...)` — creates full chain (project → epic → sprint → ticket)
- `make_test_result(project_id=...)` — for satisfying force_test_run gate

Use direct `client.post()` when you need to assert on creation behavior itself or control gate flags.

### Sprint close tests — disable ALL default-True gates
`force_handoff_md` defaults to True. When testing sprint close behavior for a specific gate, explicitly disable all others or the test will fail for the wrong reason:
```python
project = client.post("/api/projects", json={
    "prefix": "X", "name": "Test", "repo_path": str(tmp_path),
    "force_initial_md": False, "force_architecture_md": False,
    "force_handoff_md": False,
    "force_test_run": False, "force_test_coverage": False,
}).json()
```
If a new gate is added with `default=True`, sprint close tests that don't disable it will break — the 400 comes from the new gate, not the one under test.

### Response shape tests are brittle
`test_projects.py::test_get_response_shape` asserts an exact set of keys. When you add a column to the Project model, update this set or the test breaks.

### File-existence gates — use tmp_path
For gates that check file presence (`force_handoff_md` → `HANDOFF.md`, `force_initial_md` → `INITIAL.md`, etc.), use pytest's `tmp_path` fixture as the project's `repo_path`:
```python
def test_gate_passes(self, client, tmp_path):
    (tmp_path / "HANDOFF.md").write_text("# Handoff\n")
    project = client.post("/api/projects", json={
        "prefix": "X", "name": "Test", "repo_path": str(tmp_path),
    }).json()
```
Note: `force_team_md` was fully removed in DWB-321 (2026-06-05) — roster is DB-authoritative via `/api/projects/{id}/team`. The gate column is dropped; do not reference it.

### Deploy response format
The deploy-playbooks endpoint returns entries like `"pm_playbook.md"` for overwritten playbooks and `"project_rules_pm.md (created)"` for newly created project rules. Strip the suffix when checking file existence: `filename = entry.split(" (")[0]`.
